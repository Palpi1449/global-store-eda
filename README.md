# Анализ данных интернет-магазина Global Superstore

Этот проект представляет собой разведочный анализ данных (EDA) датасета Global Superstore с целью выявления поведенческих особенностей клиентов и проверки продуктовой гипотезы. Данные были загружены из CSV-файлов и обработаны с использованием Python. Проект демонстрирует навыки работы с pandas, визуализации, расчёта продуктовых метрик и формулировки выводов.

---

## 📊 Данные

В репозитории содержатся следующие файлы с данными (в папке `tables/`):

- `users.csv` – информация о клиентах (ID, имя, сегмент)
- `orders.csv` – заказы (ID заказа, ID клиента)
- `products.csv` – товары (ID товара, название, категория, подкатегория)
- `order_details.csv` – детали заказов (выручка, прибыль, даты, скидка и т.д.)
- `shipping_addresses.csv` – адреса доставки (страна, регион, город)

Данные были нормализованы из исходного CSV файла в реляционную структуру для удобства анализа.

---

## 🔍 Цель исследования

Проверить гипотезу:

 **Клиенты из сегмента `Corporate` имеют более высокий средний чек и частоту покупок, чем клиенты из сегментов `Individual` и `Home Office`.**

Для проверки гипотезы были рассчитаны следующие метрики:

- **Средний чек (AOV)** – сумма выручки в сегменте / количество заказов в сегменте
- **Частота покупок (PF)** – количество заказов в сегменте / количество клиентов в сегменте
- **Пожизненная ценность (LTV)** – сумма выручки в сегменте / количество клиентов в сегменте

## 📈 Основные этапы анализа

1. **Загрузка данных** из CSV-файлов (папка `tables/`).
2. **Предобработка**:
   - Преобразование типов (даты, числовые поля)
   - Проверка на пропуски и дубликаты
   - Проверка наличия строк с нулевой выручкой
   - Приведение сегментов к единому виду (`Consumer` → `Individual`)
3. **Анализ распределений**:
   - Гистограмма и boxplot для `sales`
   - Выявление и анализ выбросов (крупные заказы > 12 000)
4. **Корреляционный анализ**:
   - Построение матрицы корреляции Пирсона для числовых признаков с визуализацией тепловой карты.
   - Выявлена сильная положительная корреляция между `sales` и `profit` (0.74), а также между `sales` и `shipping_cost` (0.70), что подтверждает логику: дорогие заказы требуют больших затрат на доставку, но остаются прибыльными. Связи остальных признаков с выручкой и прибылью слабые.
5. **Расчёт метрик по сегментам**:
   - Средний чек, частота покупок, LTV
6. **Визуализация и сравнение**:
   - Столбчатые диаграммы с суженной осью Y для наглядности
   - Boxplot распределения сумм заказов по сегментам

## 🧪 Результаты проверки гипотезы

| Сегмент       | Средний чек (AOV) | Частота покупок | LTV (выручка на клиента) |
|---------------|-------------------|-----------------|--------------------------|
| Corporate     | 504.58            | 31.64           | 15966.45                 |
| Home Office   | 497.95            | 30.85           | 15362.46                 |
| Individual    | 503.56            | 31.63           | 15929.36                 |

**Вывод:**
- Средний чек у корпоративных клиентов (`Corporate`) **немного выше**, чем у других сегментов (разница ~1–2%).
- Частота покупок у `Corporate` и `Individual` практически одинакова, у `Home Office` чуть ниже.
- LTV у `Corporate` и `Individual` близки, у `Home Office` на 3–4% ниже.

Таким образом, **гипотеза подтверждается частично**: корпоративные клиенты демонстрируют незначительно более высокий средний чек, 
но частота и LTV сопоставимы с сегментом `Individual`. Различия статистически невелики, но могут быть значимы для бизнеса с учётом масштаба.
Теперь можно формулировать новые гипотизы для улучшения этих показателей. Например, для сегмента corporate можно предложить возможность оптовых
закупок по выгодным для обеих сторон условиям для увеличения среднего чека и частоты заказов. 
Для сегмента Individual добавить реферальную программу для роста числа клиентов и частоты покупок. А 
с сегментом home office проводить отдельные дополнительные исследования, чтобы проверить причину таких низких ключевых метрик и потом формулировать гипотезы
улучшениям для этого сегмента.
